# Developer Workspace 🚀

![Version](https://img.shields.io/badge/version-1.0.0-blue.svg)
![License](https://img.shields.io/badge/license-MIT-green.svg)

A modern, customizable developer dashboard with integrated tools, API status monitoring, and productivity features to supercharge your development workflow.

![Developer Workspace Preview](https://via.placeholder.com/800x400?text=Developer+Workspace+Preview)

## ✨ Features

- **🕒 Smart Clock Display** - Customizable clock with 12/24-hour format and timezone selection
- **🔖 Bookmark Management** - Organize development resources with customizable categories and folders
- **💻 Developer Tools** - Track GitHub activity, monitor API statuses, and access quick documentation
- **⏱️ Productivity Timer** - Built-in Pomodoro timer to help manage work/break cycles
- **📝 Developer Notes** - Terminal-style notes for quick command storage and reference
- **🌤️ Weather Widget** - Local weather information with customizable temperature units
- **🎨 Customizable UI** - Dark/light mode and multiple background options

## 🚀 Getting Started

### Prerequisites

- Node.js (v14.0.0 or higher)
- npm (v6.0.0 or higher) or yarn (v1.22.0 or higher)
- A modern web browser (Chrome, Firefox, Edge, or Safari)

### Installation

1. Clone the repository
   ```bash
   git clone https://github.com/work-rjkashyap/Developer-Workspace.git
   ```

2. Navigate to the project directory
   ```bash
   cd Developer-Workspace
   ```

3. Install dependencies
   ```bash
   npm install
   # or
   yarn install
   ```

4. Start the development server
   ```bash
   npm start
   # or
   yarn start
   ```

5. Open your browser and navigate to `http://localhost:3000`

## 📚 Usage Guide

### Dashboard Overview

The Developer Workspace dashboard is divided into several main sections:

- **Top Clock & Search** - Time display with weather widget and search functionality
- **Bookmarks Grid** - Quick access to your most important developer resources
- **Developer Panel** - GitHub activity, API status, documentation links, and timer
- **Settings Panel** - Customize appearance, clock settings, and component visibility

### Bookmark Management

1. Click the "Add" button in the bookmarks section to create a new bookmark
2. Fill in the title, URL, and select an icon or color
3. Choose a category to organize your bookmarks
4. Optionally add the bookmark to a folder for better organization

![Bookmark Management](https://via.placeholder.com/600x300?text=Bookmark+Management)

### API Status Monitoring

The Developer Workspace monitors the status of popular developer services:

- GitHub
- Cloudflare
- NPM
- Vercel
- Azure
- Netlify
- Digital Ocean
- Heroku
- CircleCI

The status is color-coded for easy visibility:
- 🟢 Green: Service is operational
- 🟠 Orange: Service experiencing degraded performance
- 🔴 Red: Service is down or unavailable

![API Status Dashboard](https://via.placeholder.com/600x300?text=API+Status+Dashboard)

### Terminal Notes

Use the terminal-style notes section to:

- Store frequently used commands
- Save quick notes and code snippets
- Access with keyboard shortcuts

Special commands:
- `clear` - Erase all stored notes
- `help` - Show available commands

## ⚙️ Configuration

### Settings Panel Options

Access the settings panel by clicking the gear icon in the top-right corner.

Available customization options:
- **Appearance**: Toggle between light and dark mode
- **Background**: Choose from numerous preset backgrounds or upload your own
- **Clock Format**: Select 12-hour or 24-hour time display
- **Time Settings**: Configure timezone and seconds display
- **Weather Units**: Toggle between Celsius and Fahrenheit
- **GitHub Integration**: Set your GitHub username for activity tracking
- **Component Visibility**: Toggle display of individual dashboard components

## 🛠️ Troubleshooting

### Common Issues

**The API status indicators aren't working**
- Ensure your browser allows cross-origin requests
- Check if your network blocks any API status endpoints
- Try refreshing the page to re-initialize connections

**Weather widget shows no data**
- Check if location services are enabled in your browser
- Ensure you've granted location permissions to the application
- Try setting your location manually in the settings
